## 🎮 5. 게임 프로젝트 목록 (Game Projects)

아래는 스터디를 통해 구현한 게임 프로젝트와 각 챕터에서 학습한 내용을 정리한 것이다.

---

### 🎮 닷지 : 3D 총알 피하기

이 챕터에서는 Unity 기초 기능을 활용해 3D 회피 게임을 완성했다.  
탄알 생성, 충돌 처리, 점수 UI, 게임 전체 흐름 관리 등 게임 개발의 기본 구조를 실습하며 전반적인 게임 제작 과정을 경험했다.

#### 📌 구현 내용

- **탄알 Prefab 생성** 및 `Instantiate()`를 통한 탄알 스폰  
- `Coroutine` + `WaitForSeconds()`로 **주기적 탄알 생성 패턴** 구현  
- `Rigidbody` 기반 플레이어 이동 및 **물리 엔진 제어**  
- `OnCollisionEnter()`로 **충돌 감지 및 게임 오버 처리**  
- TextMeshPro를 활용한 **점수 UI 표시 / 실시간 업데이트**  
- 간단한 **점수 저장 로직** 구현  
- `GameManager`를 통한 게임 흐름(스폰·점수·게임오버) 중앙 관리  
- **싱글톤 패턴**으로 GameManager 전역 접근 구조 학습  
- 씬 설정 & 빌드를 통해 **최종 게임 실행 파일 제작**  

---

#### 🛠 트러블슈팅 (Troubleshooting)

- **C#을 처음 접하면서 출력 방식과 문법이 낯설었음**  
  기존에 배웠던 언어와 구조가 달라 초기에 스크립트 작성이 생소하게 느껴졌다.

- **스크립트 오류가 자주 발생하여 해결 과정이 어려웠음**  
  오타나 참조 누락으로 오류가 많이 나서 콘솔 로그를 분석하고 원인을 찾는 과정에 시간이 걸렸지만, 덕분에 디버깅 능력이 향상됨.

- **회전 로직과 UI 배치를 이해하는 과정이 어려웠음**  
  Transform 회전 원리, EulerAngles, Canvas 구조 등 Unity 고유 개념들이 처음에는 낯설었지만 반복 실습으로 개념이 잡히기 시작함.

---

#### 💭 느낀 점 (Review / Reflection)

- Unity의 스크립트 구조가 자바의 상속 개념과 비슷해 이해에는 도움 되었지만, 상속 자체가 어렵게 느껴져 초반에 헷갈리는 부분도 많았다.  
- 만들고 싶었던 게임을 직접 구현해보며 개발 과정이 훨씬 재미있어졌고, 더 깊이 배우고 싶다는 동기부여가 생겼다.  
- 혼자였다면 오래 걸렸을 문제도 팀원 4명이 함께 해결하면서 진도를 빠르게 나갈 수 있었고, 서로 배움을 공유하며 효율적으로 학습할 수 있었다.

### 🏃 유니런 : 2D 러너

#### 📌 구현 내용

- **스프라이트 시트(Sprite Sheet)를 활용해 여러 이미지를 하나의 스프라이트로 다루는 방법 학습**
  - 러닝·점프·착지 등의 여러 이미지가 포함된 스프라이트 시트를 분리해 사용  
  - `Sprite Editor`로 슬라이스 처리 및 애니메이션 연결  

- **오디오 소스(AudioSource)를 이용한 사운드 재생**
  - 점프, 착지, 충돌 등 상황에 따라 효과음 재생  
  - BGM과 SFX를 분리하고 Inspector에서 편하게 조절하도록 구성  

- **유한 상태 머신(FSM, Finite State Machine) 활용**
  - 캐릭터의 상태(달리기, 점프, 이단 점프, 낙하 등)를 명확히 나눔  
  - 상태에 따라 어떤 애니메이션을 재생할지 결정하는 로직 구현  

- **애니메이터(Animator) & 애니메이터 컨트롤러(Animator Controller) 이해**
  - 각 상태(State) 간 전이(Transition) 조건 설정  
  - 파라미터(Trigger, Bool, Float)를 기반으로 자연스러운 애니메이션 전환 구현  

- **이단 점프(Double Jump) 기능 구현**
  - 점프 가능 횟수를 변수로 관리  
  - 공중 상태에서 추가로 점프할 수 있도록 로직 구성  

- **바닥 감지(Ground Check) 로직 구현**
  - Raycast 또는 Collider를 활용해 플레이어가 바닥 위에 있는지 판정  
  - 바닥 여부에 따라 점프 초기화 및 애니메이션 전환 처리  

---

#### 🛠 트러블슈팅 (Troubleshooting)

- **유니티 공간(좌표·벡터) 개념이 유니런 개발과 직접 연결되었지만 이해가 어려웠음**  
  유니런을 만들기 전 Unity 공간과 좌표계를 공부했는데, 이 개념이 실제 플레이어 이동·점프·발판 이동 등에 모두 연관되어 있었다.  
  특히 `Vector2`, `Vector3` 개념을 이해하는 데 시간이 오래 걸렸고, 아직도 완전히 익숙하지 않아 추가 학습이 필요하다고 느낌.

- **발판(Platform)이 움직이지 않는 오류 발생**  
  이동 로직을 올바르게 작성했다고 생각했음에도 발판이 움직이지 않는 문제가 생겼다.  
  Rigidbody 설정·Update와 FixedUpdate 구분·Inspector 값 등을 확인하며 여러 요소를 점검했고, 이를 통해 Unity 물리 시스템의 특징을 더 깊이 이해하게 되었다.

- **가시(Spike)가 랜덤으로 생성되지 않고 계속 반복 생성되는 버그**  
  `0, 3 == 0` 같은 조건식 구성에는 문제가 없었지만, **정작 중요한 스폰 로직 일부가 빠져 있어서**  
  랜덤 함수가 의도대로 적용되지 않았고, 그 결과 가시가 연속적으로만 생성되는 현상이 발생했다.  
  콘솔에 오류가 뜨지 않아 원인 찾기가 특히 어려웠고, 코드를 단계적으로 주석 처리하며 문제를 좁혀가는 방식으로 해결했다.

---

#### 💭 느낀 점 (Review / Reflection)

- 오류가 발생했을 때 당황하지 않고 **콘솔 메시지·라인 번호·오류 유형을 확인하며 문제를 추적하는 능력**이 크게 향상되었다.  
  이를 통해 전체적인 C# 문법·로직·흐름을 더 깊게 이해하게 된 계기가 되었다.

- 코드 자체에는 문제가 없었지만, **중요한 코드가 빠져 있을 때는 콘솔 오류도 나오지 않아 디버깅이 더 어려울 수 있다는 점**을 경험했다.  
  이런 상황에서는 팀원들과 디스코드를 통해 함께 코드를 리뷰하며 분석했고, 그 과정에서 문제 해결력과 협업 능력을 동시에 기를 수 있었다.

- 서로의 코드를 비교하며  
  “이 부분이 왜 이렇게 작동하지 않지?”  
  “이 조건이 이렇게 되면 어떤 결과가 나오는구나”  
  같은 식으로 상호 피드백을 주고받으며 학습한 것이 큰 도움이 되었다.  
  혼자서 해결하기 어려운 문제도 팀원들과 머리를 맞대면 빠르게 해결되는 경험을 하면서,  
  **협업의 장점과 코드 리뷰의 중요성을 직접 체감할 수 있었다.**
